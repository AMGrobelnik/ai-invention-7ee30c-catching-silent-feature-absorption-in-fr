# First-Letter-Spelling Absorption Testbed

The load-bearing dataset for the Two-Track Co-Response Grouping hypothesis (reliable cluster-level SAE
units). **Pure CPU/data artifact** — no SAE, no model weights, no GPU. It anchors words in the real
**gemma-2-2b** vocabulary and the **Pile** so it is non-circular (words never come from the SAE latents
being grouped), guaranteed-signal (first-letter is the documented absorption regime), and frozen
(pinned tokenizer + pinned Pile revision).

## Files
- `full_data_out.json` — 17,180 examples in **5 per-letter dataset groups** (`first_letter_spelling_{L,O,T,I,D}`), ~21 MB (schema `exp_sel_data_out`, PASSED).
- `mini_data_out.json` / `preview_data_out.json` — generated by the aii-json format script (mini: 3 examples per dataset group; preview: truncated, strings ≤200 chars). Both schema-PASSED.
- `data.py` — full reproducible builder (`uv run data.py`, deps pinned in `pyproject.toml`). `make_variants.py` — alternate variant generator.
- `temp/datasets/SOURCES.md` — source evaluation & provenance.

## What it provides (3 components per target letter; L primary, then O/T/I/D)
- **(A) content_flip** — 1,750 pairs (3,500 rows). `(x_on,x_off)` in an identical carrier; `x_on` slots a
  word **starting with the target letter**, `x_off` a surface-matched word that does **not** (matched on
  char length, single-token-ness, and Pile log-frequency). For the Tier-0 K-proposal pilot and the C3
  absorber-recovery spine. Compute `r_l = a_l(x_on) − a_l(x_off)`.
- **(B) surface_flip** — 590 pairs (1,180 rows). `(x_a,x_b)` in an identical carrier; **both** words start
  with the target letter but differ (case-insensitively). Concept held constant, surface varies → the
  unit-level surface-invariance admission check (a good unit's pooled response to surface flips ≈ 0).
- **(C) corpus_context** — 12,500 rows (2,500/letter). Real ~48-token windows from `monology/pile-uncopyrighted`
  @ rev `3be9033`, each centred on a slot-eligible word-initial target-letter token, with its position
  annotated. Plus a per-letter occurrence table (≤2,000 word-types) in dataset metadata → substrate for
  iteration-2's form-free/Chanin (2409.14507) diagnostic to find absorbed false-negatives ('lion','London').

## Carriers (7 per content pair)
`t_verbose` (`{word} has the first letter:` — sae-spelling VERBOSE_FIRST_LETTER_TEMPLATE), `t_colon`
(`{word}:`), `t_icl` (4 shuffled, contamination-free ICL examples + verbose query), and 4 word-class-agnostic
mention carriers (`The word {word} appears in the text.` etc.). Mention carriers replace noun-assuming
sentences because frequency-ranked on-words are mixed POS.

## Row schema (flat, exp_sel_data_out-compliant)
`input` (text the SAE encodes), `output` (first letter, uppercase), and `metadata_*`:
`dataset, letter, pair_id, pair_type, role, sub_context` (the word covered — the key field),
`target_word, counterpart_word, template_id, label_starts_with_target, is_single_token, is_slot_eligible,
first_letter, fold`. Pairs add `word_char_span` (indexes `input`). Corpus rows add `source_doc_id,
pile_revision, token_position, target_token_id, window_char_span, target_char_in_window, judge_pass`.
Pairs LINK via shared `metadata_pair_id` + `metadata_role` ({on,off} / {var_a,var_b}).

## Usage contract (also in `metadata.generation_config.usage_contract`)
- **Pairs**: group by `metadata_pair_id`; `word_char_span` indexes the slotted word in `input`.
- **Corpus**: `token_position` is the target-token index in `tok(input, add_special_tokens=False)`;
  **verified exact on 400/400 sampled rows** (`ids[token_position]==target_token_id`).
- **Folds**: pairs by `target_word` (no word-identity leakage), corpus by `source_doc_id` (no doc leakage); 5 folds via md5-hash.

## Validation
- **Deterministic (authoritative): 0 violations / 17,180 rows** — flip property + span correctness are
  guaranteed by construction (first letters taken straight from the tokenizer); violations would be dropped.
- **LLM judge (secondary grammaticality/independent audit, `google/gemini-3.1-flash-lite`, $0.12)**:
  pass rates **0.89–0.99** per (letter,pair_type); `judge_false_negative_rate` quantifies judge slips on
  this short-prompt task (survivors already pass the deterministic check, so the judge never drops pairs).

## Reproduce
```
python data.py --letters l,o,t,i,d --n_docs 20000 --judge_per 150 \
  --judge_model google/gemini-3.1-flash-lite --out full_data_out.json
```
Pins: tokenizer `unsloth/gemma-2-2b` (vocab 256000), Pile `monology/pile-uncopyrighted`
rev `3be90335b66f24456a5d6659d9c8d208c0357119`, seed 1234. Total LLM spend: **$0.12** (< $3 cap).
